온톨로지 없이는 신뢰할 수 없다:
AI 에이전트 시대의 시맨틱 인프라 논쟁

Semantic Brief 편집팀
2026년 4월 7일
Vol.97

2026년 봄, 데이터 인프라 업계에서 가장 뜨거운 논쟁이 벌어지고 있다. 그 핵심은 단순하지만 치명적인 질문이다. "AI 에이전트가 쿼리를 생성할 때 과연 기업의 데이터를 올바르게 이해하고 있는가?" dbt Labs와 Snowflake, Cube가 치열하게 경쟁하는 시맨틱 레이어 시장에서, 그리고 Microsoft의 GraphRAG이 엔터프라이즈 RAG의 표준으로 부상하는 흐름 속에서, '온톨로지'라는 오래된 기술이 AI 인프라의 새 토대로 재조명받고 있다.

전통적으로 온톨로지는 철학적·학문적 영역의 개념으로 여겨졌다. 시맨틱 웹의 꿈을 좇던 W3C의 RDF, OWL 표준들은 실무 현장에서 "너무 복잡하다"는 이유로 오랫동안 외면받았다. 하지만 LLM이 실제 기업 데이터에 접근하게 되면서, 그리고 AI 에이전트가 비즈니스 의사결정을 보조하게 되면서, 데이터의 '의미'를 명시적으로 정의하지 않으면 치명적인 오류가 발생한다는 사실이 드러나기 시작했다. SQL 쿼리 자동 생성에서 발생하는 10~30%의 오류율, 지표 정의의 불일치, 데이터 계보의 불투명함—이 모든 문제의 근원에 시맨틱 정의의 부재가 있다.

"온톨로지 기반 시맨틱 레이어는 LLM이 필요로 하는 구조를 제공한다. 왜냐하면 그것이 LLM이 훈련된 방식, 즉 명시적인 비즈니스 개념과 정의된 연결 관계와 일치하기 때문이다."

— Timbr.ai, 'Why Snowflake Needs an Ontology-Based Semantic Layer', 2026

이번 주 가장 상징적인 사건은 Digital Science의 Ontopic 인수다. Bolzano 자유대학의 스핀오프인 Ontopic은 오픈소스 VKG(Virtual Knowledge Graph) 프레임워크 Ontop의 개발사로, 수십 년에 걸쳐 쌓아온 OBDA(Ontology-Based Data Access) 연구를 실용화한 회사다. Digital Science는 이 인수를 통해 Ontopic Studio와 자사의 metaphactory 플랫폼을 결합, "데이터 변환 없이 기존 소스에서 직접 지식 그래프를 구축하고 쿼리하는" 엔드투엔드 파이프라인을 완성한다고 밝혔다. 단순한 M&A를 넘어, 가상 지식 그래프—즉, 물리적 데이터 이동 없이 온톨로지 매핑으로 의미 계층을 구성하는 기술—가 드디어 엔터프라이즈 시장의 주류에 편입되는 신호탄이다.

넷플릭스의 QCon London 2026 발표도 이 흐름을 방증한다. 넷플릭스는 사용자, 클라이언트, 서비스, 인프라를 연결된 그래프로 모델링하는 대규모 E2E 지식 그래프를 온톨로지 기반으로 구축했고, 이를 통해 '온톨로지 기반 관찰 가능성(Ontology-Driven Observability)'이라는 개념을 실제 넷플릭스 스케일에서 구현했다. 이것이 의미하는 바는 크다. 온톨로지는 더 이상 학술 논문 속의 개념이 아니라, 초대형 기술 기업의 핵심 운영 인프라가 된 것이다.

그러나 논쟁은 여기서 끝나지 않는다. dbt Semantic Layer, Snowflake Semantic Views, Cube, Timbr—각자는 서로 다른 철학적 토대를 갖고 있다. dbt와 Snowflake가 SQL 중심의 실용적 접근을 취한다면, Timbr는 온톨로지 추론(reasoning)과 계층적 개념 상속을 전면에 내세운다. 2025년 등장한 Open Semantic Interchange(OSI) 이니셔티브가 이 분열된 생태계를 통합할 수 있을지는 아직 미지수다. 한 가지는 분명하다. 시맨틱 인프라를 먼저 견고하게 구축한 기업이, 다가올 AI 에이전트 시대에서 신뢰할 수 있는 데이터 기반을 가진 유일한 플레이어가 될 것이다.

03 · 기술 딥다이브

GraphRAG의 해부:
단순 벡터 검색을 넘어선 지식 그래프 기반 RAG

2024년 Microsoft Research가 발표한 GraphRAG은 2026년 현재 엔터프라이즈 RAG의 새 표준으로 자리매김하고 있다. 기존 벡터 유사도 기반 검색(Naive RAG)이 갖는 구조적 한계—맥락 단절, 글로벌 테마 파악 불가, 복합 추론 불가—를 지식 그래프 구조로 극복하는 이 접근법을 깊이 들여다본다.

GraphRAG이란 무엇인가

GraphRAG(Graph Retrieval-Augmented Generation)은 비정형 텍스트 데이터셋으로부터 LLM을 활용해 지식 그래프를 자동 구축하고, 쿼리 시점에 그래프 머신러닝과 결합해 프롬프트를 강화하는 엔드투엔드 시스템이다.

핵심 차이: 일반 RAG이 "관련 문서 조각"을 검색한다면, GraphRAG은 "엔티티 간 관계와 커뮤니티 요약"을 검색한다. 이는 "A사의 CEO는 어느 대학 출신인가?"같은 단순 검색은 물론, "이 문서 컬렉션에서 주요 테마와 논쟁점은 무엇인가?"같은 글로벌 분석도 가능하게 한다.

그림 1. GraphRAG 아키텍처: 인덱스 구축(좌) → 지식 그래프(중) → 쿼리 파이프라인(우)

로컬 검색 vs 글로벌 검색

로컬 검색(Local Search): 쿼리의 엔티티와 직접 연결된 서브그래프를 검색한다. "삼성전자의 주요 파트너사는?"처럼 특정 엔티티 중심의 질문에 최적화된다. 벡터 유사도 + 그래프 트래버설을 결합해 관련 텍스트 청크와 관계를 함께 반환한다.

글로벌 검색(Global Search): 전체 문서 컬렉션에서 주요 테마·패턴을 파악하는 분석에 사용된다. 커뮤니티 요약을 맵-리듀스 방식으로 처리해 "이 보고서 전체에서 가장 자주 논의되는 리스크 요인은?"같은 질문에 답한다. 비용이 높지만 기존 RAG으로는 불가능한 글로벌 인사이트를 제공한다.

예시 코드 · Python — FalkorDB + GraphRAG SDK 기본 파이프라인
# pip install graphrag-sdk falkordb
from graphrag_sdk import KnowledgeGraph, Ontology
from graphrag_sdk.models.openai import OpenAiGenerativeModel
from falkordb import FalkorDB

# 1. 온톨로지 정의 — 엔티티·관계 스키마
ontology = Ontology()
ontology.add_entity("Company", properties=["name", "founded", "sector"])
ontology.add_entity("Person",  properties=["name", "role", "expertise"])
ontology.add_entity("Technology", properties=["name", "maturity"])
ontology.add_relation("EMPLOYS",   src="Company", tgt="Person")
ontology.add_relation("USES",      src="Company", tgt="Technology")
ontology.add_relation("DEVELOPS",  src="Person",  tgt="Technology")

# 2. 지식 그래프 구축 — FalkorDB 백엔드 사용
model = OpenAiGenerativeModel(model_name="gpt-4o")
db = FalkorDB(host="localhost", port=6379)

kg = KnowledgeGraph(
    name="enterprise_kg",
    ontology=ontology,
    model=model,
    graph=db.select_graph("enterprise_kg")
)

# 3. 문서 인덱싱 — 자동 엔티티·관계 추출
with open("documents/annual_report_2025.txt") as f:
    kg.process_sources([f.read()])

# 4. 쿼리 — 자연어 → 그래프 증강 응답
chat_session = kg.chat_session()

# 로컬 쿼리: 특정 엔티티 중심
response_local = chat_session.send_message(
    "삼성SDS가 활용하는 주요 AI 기술과 담당 인력을 알려줘."
)

# 글로벌 쿼리: 전체 문서 테마 분석
response_global = chat_session.send_message(
    "이 보고서 전체에서 가장 자주 언급되는 리스크 요인 TOP3는?"
)

print(response_local["response"])
print(response_global["response"])

2026년 GraphRAG의 현재와 한계

성과: 엔터프라이즈 도입이 확산되면서 고객 지원 시스템에서 평균 29.6% 빠른 해결 시간을 보고하는 사례가 등장했다. Microsoft GraphRAG 프레임워크는 ICLR'26에서 발표된 LinearRAG 방법론 등 후속 연구를 지속적으로 흡수하며 성숙하고 있다.

한계: 실시간 업데이트의 어려움과 초대형 데이터셋 확장성이 여전히 주요 과제다. 초기 인덱스 구축 비용(LLM 호출 비용)이 높고, 커뮤니티 탐지의 품질이 최종 응답 품질에 직접 영향을 미친다. 온톨로지 설계의 품질이 전체 파이프라인 성능을 좌우하므로, 도메인 전문가와 데이터 엔지니어의 협업이 필수다.

04 · 도구 비교

2026년 시맨틱 레이어 스택 선택 가이드

시맨틱 레이어 시장은 2026년 현재 크게 세 진영으로 나뉜다. Warehouse-Native(Snowflake Semantic Views, BigQuery BI Engine), SQL 기반 독립 레이어(dbt MetricFlow, Cube), 그리고 온톨로지 기반 엔터프라이즈(Timbr, Ontop/Ontopic)가 그것이다. 각각의 장단점과 이상적인 유즈케이스를 비교한다.

도구 오픈소스 클라우드 지원 KG 연동 AI/NL2SQL 러닝커브 비용 구조

유즈케이스별 추천 스택

Snowflake 올인 스타트업

이미 Snowflake를 데이터 플랫폼으로 사용하며, 빠른 BI 지표 정의와 Cortex AI 연동이 필요한 스타트업

→ Snowflake Semantic Views + dbt MetricFlow

멀티 웨어하우스 데이터팀

Snowflake·BigQuery·Databricks를 혼합 사용하며, 통합 시맨틱 레이어와 강력한 API 지원이 필요한 팀

→ Cube OSS (Self-hosted) + Cube Cloud

AI 에이전트 도입 엔터프라이즈

LangChain 기반 AI 에이전트를 기업 데이터에 연결하고, NL2SQL 정확도와 감사 추적이 필수인 대기업

→ Timbr + LangChain SDK

레거시 DB 시맨틱화 (VKG)

수십 년 된 관계형 DB를 데이터 마이그레이션 없이 의미 계층으로 감싸야 하는 공공기관·제조업

→ Ontop OSS + Ontopic Studio

GraphRAG 파이프라인 구축

내부 문서·계약서·보고서에서 지식 그래프를 구축하고, 복합 추론 기반 AI Q&A를 제공하려는 팀

→ FalkorDB + GraphRAG-SDK + Timbr(시맨틱 레이어)

학술·연구 기관

RDF/OWL 기반의 도메인 온톨로지(생물·의료·과학)를 SPARQL로 쿼리하고 지식 그래프와 연동해야 하는 연구 기관

→ Ontop + GraphDB + SPARQL Endpoint

2026년 선택의 핵심 기준

AI 에이전트 우선이라면 온톨로지 기반 접근이 필수다. LLM이 정확한 SQL을 생성하려면 데이터의 의미가 명시적으로 정의되어 있어야 한다. 단순 SQL 메트릭 정의(dbt, Snowflake Views)는 BI 쿼리에는 충분하지만, 자유 형식 NL2SQL에서는 한계를 드러낸다.

비용 최적화를 원한다면 Warehouse-Native부터 시작하라. Snowflake Semantic Views는 추가 인프라 비용 없이 즉시 시작할 수 있다. 이후 온톨로지 기반으로 마이그레이션하는 경로를 설계하는 것이 현실적이다.

Open Semantic Interchange(OSI)를 주목하라. dbt Labs, Snowflake, Salesforce가 참여하는 표준화 이니셔티브가 성숙하면, 특정 벤더에 대한 락인 없이 시맨틱 정의를 이식할 수 있게 된다. 지금은 학습 비용이 낮은 솔루션을 선택하되, OSI 호환성을 장기 체크리스트에 포함시켜야 한다.

핵심 용어 정리

Ontology
도메인 내 개념(클래스)과 그 관계, 속성을 명시적으로 정의한 형식적 지식 표현. RDF/OWL 표준 기반. 기계가 추론할 수 있는 구조적 의미 모델.
VKG (Virtual KG)
기존 데이터 소스(RDB, CSV 등) 위에 온톨로지 매핑을 씌워 데이터 이동 없이 지식 그래프처럼 쿼리하는 기술. Ontop이 대표 구현체.
GraphRAG
텍스트에서 지식 그래프를 구축하고, 쿼리 시 그래프 구조를 활용해 RAG의 응답 품질을 높이는 기법. Microsoft Research 주도 오픈소스 프레임워크.
Semantic Layer
데이터 웨어하우스와 BI/AI 도구 사이에서 비즈니스 지표와 개념의 정의를 중앙화하는 추상화 계층. MetricFlow, Cube, Snowflake Semantic Views 등이 구현체.
OBDA
Ontology-Based Data Access. 온톨로지를 통해 이기종 데이터 소스에 통합 접근하는 패러다임. SPARQL 쿼리를 SQL로 변환해 실행하는 것이 핵심.
NL2SQL
자연어(Natural Language) 질문을 SQL 쿼리로 자동 변환하는 기술. 시맨틱 레이어와 결합할 때 정확도가 크게 향상됨. 엔터프라이즈 AI 에이전트의 핵심 기능.
